# 4. 훈련 결과
## 가. 기업 경영 이슈 (As Is)
- [AI 활용 역량 부족]  
  회사 내에서 AI 기반 서비스 기획·개발 경험이 적어, 실제 서비스에 LLM을 도입하는 과정에서 기술적 이해 부족 및 적용 전략 부재 문제가 존재함.
- [기존 응용 개발과 프로세스 차이 극복 필요]  
  기존 개발 방식 중심의 사고방식이 강해 LLM 기반 서비스 개발 프로세스(프롬프트 설계, 모델 한계 관리, 데이터 품질 등)에 대한 이해도가 낮아, 시나리오 설계 및 리스크 대응이 어려움.
- [데이터 활용 미흡]  
  서비스에 존재하는 데이터는 있으나, LLM 서비스 평가·테스트에 활용 가능한 형태로 정제·구조화된 데이터가 무엇인지 식별에 어려움을 겪음. 신규 서비스 설계 시 정확한 성능 판단이 어려움.
- [AI 도입에 대한 낮은 구체성]  
  내부적으로 “AI 도입 필요성”은 알고 있으나, 실제 어떤 방식으로 서비스에 통합할지, 어떤 아이템이 비즈니스 임팩트가 있는지 명확한 기준이 부족함.

## 나. 개선 결과 (To Be)
- [ LLM 작동 원리 및 한계 이해도 향상 ]  
  학습을 통해 LLM의 구조, 추론 방식, 한계(환각, 일관성 문제, 문맥 단절 등)를 명확히 이해하게 되었으며, 이를 보완하기 위한 프롬프트 엔지니어링·RAG·Tool Calling 사용 전략을 습득함.
- [ 서비스 적용 가능성 발굴 성공 ] 
  자사 기존 서비스 분석을 기반으로, 실제로 적용 가능한 AI 활용 포인트(AI 데이터 처리 대상선정, 응답 품질 향상을 위한 보완 방법 선정)를 도출하고 시나리오 초안을 완성함.
- [ 프로토타입 수준의 구현 가능성 확인 ]
  선정된 시나리오 일부를 프로토타입 수준의 실험 워크플로우 형태로 동작 가능성을 검증했으며, 프롬프트 설계 및 모델 출력 품질 개선 경험 확보.
- [ 개발시 로직컬 구현 영역과 AI 연계 구현 영역 분리 가능 ]
  향후 PoC 및 실서비스 도입 시 평가 체계를 설계할 수 있는 수준까지 역량이 향상됨.
- [ AI 서비스 기획 역량 자체 고도화 ]
  단순 아이디어 제시 수준이 아닌 → 분석 → 시나리오 설계 → 테스트/평가 → 고도화 → 구현점 검증의 과정을 통해 AI 활용 프로젝트 리딩 역량이 향상됨.
